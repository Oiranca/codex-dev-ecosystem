# Agent Decisions Log

> Shared decision log for milestone-based agent execution. Auto-maintained during repository work. Do not delete.

## Usage
- Record one concise entry per execution cycle.
- Keep entries focused on the active milestone only.
- Note what was executed, what was skipped, and why.
- Preserve history; append new entries instead of rewriting old ones.

## Entry Template

## [YYYY-MM-DD HH:MM] Cycle N
- **Objective**: <current goal>
- **Milestone**: <active milestone or NONE>
- **Executed**: <agents, skills, or validations run>
- **Skipped**: <agents, skills, or validations skipped and why>
- **Risks / Blockers**: <key blockers or NONE>
- **Result**: SUCCESS | PARTIAL | BLOCKED | SKIP
